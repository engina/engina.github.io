suphp SoftException in Application.cpp:299: Script "foo" resolving to "foo" not within configured docroot

07 Nov 2012

Recently I've switched to suphp from regular mod_php to support one application (wordpress) better and for obvious security benefits. Though, one of my application worked flawlessly, another one failed with the following error message in the log.

Actual error:

[Wed Nov 07 19:35:27 2012] [error] [client 78.171.237.162] SoftException in Application.cpp:299: Script "/home/engin/www/api.php" resolving to "/home/engin/www/api.php" not within configured docroot

It's easy to think that this is related to Apache vhost configuration's DocumentRoot but it is not.

My application which is working was located in /home/usera/public_html and the failing one was in /home/userb/www. Now check the /etc/suphp/suphp.conf's docroot variable:

docroot=/var/www:${HOME}/public_html

As you see, ${HOME}/public_html is in the docroot, hence one of my application was working and the other one was not. I moved www to public_html and all is fixed.
