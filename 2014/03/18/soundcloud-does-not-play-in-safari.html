Soundcloud does not play in Safari [FIXED]

18 Mar 2014

Soundcloud stopped working in Safari for some time now. Today when I couldn’t listen to a track shared via Facebook, I had enough and decided to take a closer look.

A quick googling revealed that this is a known problem due to new power saving features implemented in Safari. Though, I didn't like the workaround presented in the support page because Flash plugins are indeed a CPU and battery hog and I want Safari to save power on them.

With this new feature, Safari does not automatically start plugins unless it is sure that the content is safe or you explicitly click on the plugin "box" to allow it. For video sites, this is not much of a problem, you click on the giant plugin box which says “Click to run the plugin” to start the plugin manually. But as with sites such as Soundcloud, the hidden flash player is a problem. There’s nowhere to click and explicitly allow the plugin to run! All the player controls you see are HTML interface to the Flash backend.

In the preferences you can see a list of sites which the plugins will automatically start working. Unfortunately though, there are no ways to manually add sites to that list. You have to go through the above process and manually click to start a plugin once, so that Safari will add it to a list.

[caption id="attachment_800" align="aligncenter" width="668"] You can see the list of web sites that Safari will allow plugin execution automatically for by clicking Details button.[/caption]

[caption id="attachment_799" align="aligncenter" width="668"] As you can see there are no user interface to manually add web pages here.[/caption]

So, I could either:
  1. Hack through Safari config files and add soundcloud.com to that exception list externally.
  2. Make the Flash plugin visible so I can click on it.
Obviously the second approach seems less risky as you can’t screw up configuration files. Here are step by step instructions:
1. Open soundcloud.com in Safari and let it load.
2. Try to play the track. This will make sure the player is injected in the page so we can perform next step.
3. Make sure your "Show Develop Menu in menu bar" is checked in Preferences>Advanced.
4. Open the Web Inspector from the Develop menu.
5. Then execute the following by copy pasting it into the textfield at the bottom of the Web Inspector. You can edit the first 512 to adjust the size of the box that'll (hopefully) appear on your screen.
var w=512; $('#flashAudioObject').css({width:w, height:w, maxWidth:w, maxHeight: w}).parent().css({position:'absolute', top: '50%', left: '50%', marginTop:-w/2, marginLeft:-w/2, width:w, height: w, zIndex:999999})

6. A black box should appear in the center of your screen. Click in it a few times just to make sure.

7. You should be able to play the track now and soundcloud.com will be listed in the exceptions listed mentioned above.

That's it.

 

Conclusion:

IMHO, the best solution for SoundCloud would be to show a one time modal popup to users who are affected by the problem, guiding them to click on a (congratulations?) video to enforce this workaround.

At the moment, users have to manually google the problem, then use an unideal workaround as fix.
