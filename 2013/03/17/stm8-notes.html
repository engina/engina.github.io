STM8 Notes

17 Mar 2013

STM8
- program memory bus 32-bit, can fetch 1 or 2 instructions at once
- instruction pipeline: fetch, decode, execute, write operations overlap for the next instruction, so while decoding n-th instruction, CPU is fetching n+1-th instruction and so on.
- stack is decremented
- timer interrupt can be as late as 15 cycles. executing instruction can take 1-6 cycles, and context save is fixed 9 cycles.
- 0xXX0000 - 0xXXFFFF is a section where XX is section number
- 0xXXXX00 - 0xXXXXFF is a page where XXXX is page number
- CALL, CALLR, RET can be used in the same section only. This stores PH and PL into the stack. !!NOT VERY CLEAR section 4.1.!!
- JPF, CALLF, RETF can be used to jump to any place -- F is far. This stores whole PE, PH, PL - 24-bit PC into stack.
- As SP is 16-bit, it has to be in the section 0.
- Interrupt vector is at 0x8000 with 32 4 byte data. Where MSB is reserved 0x82 opcode and the remaining 24-bit is the address of the interrupt routine
- STM8 uses Hardware Architecture and has two busses; program and data memory.
- STM8 has a 3 stage pipeline Fetch, Decode, Execute. PC always points to the instruction in the decode stage.
- Any instruction bigger than 4-bytes requires 2 fetches
- Flash fetches are 4-byte aligned
- Flash bus is 32 bit where RAM bus is 8 bit.. Hence program execution from RAM is slower.
- Opcode is 1-2 byte, data/operand 0-3 bytes
- reg-to-reg operation can be 1 byte in total... LD A, XH is 0x95 as opcode and no data/operand at all.
- if an instruction is spread over an alignment boundary, two fetches will be required
- for indirect access CPU is stalled during fetching the pointer value from memory
- if two stages (decode, execute) tries to access to same memory region, decode is halted until execution releases the memory resource.
- Jumps flush pipeline
